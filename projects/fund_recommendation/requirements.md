# 用python语言实现一个基金信息管理分析系统。

# 主要功能：

## 基金数据库， 包括以下几张表。悬着的数据库产品需要免费，轻量，并且支持比较强大的查询分析功能，比如支持复杂的join查询，聚合函数，窗口函数等。
1.  基金基本信息表（fund_basic_info）
包含字段：代码，名称，基金类型，份额类型(A份额或者其他份额)，主基金代码（如果不是A份额的基金，对应的A份额基金代码），成立日期，上市日期，基金管理人，基金托管人，管理费率， 托管费率， 销售服务费率，业绩比较基准，跟踪标的，更新日期

2. 基金资产规模表（fund_asset_scale）
包含字段：代码，资产规模（单位：亿元），份额规模（单位：亿份），更新日期

3. 基金经理表（fund_manager）
包含字段：代码，基金经理姓名，任职开始日期，任职结束日期，更新日期

4. 基金申购状态表（fund_purchase_status）
包含字段：代码，申购状态（是否可申购），申购手续费（单位：元），更新日期

5. 基金赎回状态表（fund_redemption_status）
包含字段：代码，赎回状态（是否可赎回），赎回手续费（单位：元），更新日期

6. 基金历史净值表（fund_nav_history）
包含字段：代码，日期，净值，净值增长率，更新日期

7. 基金业绩表（fund_performance）
包含字段：代码，周期（包括成立以来，今年以来，历史年份），净值增长率，最大回撤，下行标准差，夏普比率，索提诺比率，卡玛比率，年化收益率，波动率，同类型基金排名，更新日期

## 数据获取模块，从akshare数据源获取基金的数据处理并更新数据库。
注意点：
1. 获取基金净值数据时加入随机延时，增加成功率。同时加入并发和本地缓存机制，避免重复获取数据，提高效率。
2. 该模块可以人工运行或者定时运行，默认每天运行一次
    1. 基金基本信息表（fund_basic_info）主要是增加新基金
    2. 基金资产规模表（fund_asset_scale）主要是更新基金的资产规模和份额规模，对于资产规模或者份额规模的变化，需要新加入记录。
    3. 基金经理表（fund_manager）主要是更新基金经理的信息，对于基金经理的变化，需要新加入记录。
    4. 基金申购状态表（fund_purchase_status）主要是更新基金的申购状态和手续费，对于申购状态或者手续费的变化，需要新加入记录。
    5. 基金赎回状态表（fund_redemption_status）主要是更新基金的赎回状态和手续费，对于赎回状态或者手续费的变化，需要新加入记录。
    6. 基金历史净值表（fund_nav_history）主要是更新基金的历史净值数据，对于新日期的净值数据，需要新加入记录。
    7. 基金业绩表（fund_performance）主要是更新基金的业绩指标，历史年份的数据不用计算（比如2020年的净值增长率，最大回撤等指标），只需要计算成立以来和今年以来的指标，并加入记录。
2. akshare接口使用提示：
    1. ak.fund_name_em()可以拿到所有基金的代码，名称和基金类型
    2. ak.fund_overview_em()可以拿到指定基金的概况
    3. ak.fund_fee_em()可以拿到指定基金的交易费率
    4. ak.fund_open_fund_daily_em()可以拿到所有基金当日最新净值数据
    5. ak.fund_open_fund_info_em(symbol=code, indicator="单位净值走势")可以拿到指定基金的历史净值数据

## 基金分析推荐模块，用于根据基金业绩表和基金历史净值表分析基金的表现和风险，推荐基金。
该模块需要支持添加各种分析处理逻辑，目前主要包括：
1. 根据指定的基金类型，筛选出在指定的时间范围内净值增长率Top-N和bottom-N的基金, 比如用户指定基金类型是混合型，时间范围是2020-01-01到2020-08-31，N是20，那么就筛选出在2020-01-01到2020-08-31区间Top-10和Bottom-10的混合型基金
2. 根据指定的基金类型，时间范围和分析间隔，筛选逻辑：
    1. 首先将指定的时间范围按照分析间隔进行划分，得到多个分析间隔，比如用户指定时间范围是2020-01-01到2023-12-31，分析间隔是1年，那么就得到[2020-01-01，2020-12-31], [2021-01-01，2021-12-31], [2022-01-01，2022-12-31],[2023-01-01，2023-12-31]这4个分析间隔。
    2. 对于每个分析间隔，筛选出在该时间范围内净值增长率Top-N的基金，比如用户指定基金类型是混合型，时间范围是2020-01-01到2020-08-31，N是20，那么就筛选出在2020-01-01到2020-08-31区间Top-10的混合型基金
    3. 对于每个分析间隔，筛选出在该时间范围内净值增长率Bottom-N的基金，比如用户指定基金类型是混合型，时间范围是2020-01-01到2020-08-31，N是20，那么就筛选出在2020-01-01到2020-08-31区间Bottom-10的混合型基金
    4. 根据用户指定的%X比例，筛选出在%X比例的分析间隔内净值增长率都在Top-N的基金, 比如用户指定时间范围是2020-01-01到2023-12-31，分析间隔是1年，%X是50%，N是10，那么就筛选出[2020-01-01，2020-12-31], [2021-01-01，2021-12-31], [2022-01-01，2022-12-31],[2023-01-01，2023-12-31],这4个分析间隔中任意两个分析间隔内净值增长率都在Top-10的基金
    5. 对于每个分析间隔，分别各处相比前一个分析间隔净值增长率Top-N的基金，比如用户指定时间范围是2020-01-01到2023-12-31，分析间隔是1年，%X是50%，N是10，那么就筛选出[2020-01-01，2020-12-31], [2021-01-01，2021-12-31], [2022-01-01，2022-12-31],[2023-01-01，2023-12-31]这4个分析间隔中，每个分析间隔相比上一个间隔的净值增长率都在Top-10的基金。
3. 基于千问给出的多因子量化评估，在给定的基金类型下选出Top-T的基金，输出结果保存到CSV文件中。具体逻辑参考.\fund_recommendation_multiple_factors.py文件中的实现。
