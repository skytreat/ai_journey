# 【规则名称】Trae Python 编码规范（全局生效）
## 规则基础信息
- 规则ID：trae_python_2026
- 生效范围：全局
- 违规级别：核心约束为 error，注释约束为 warning
- 适用场景：数据分析/后端开发

## 核心约束项
### 1. 基础格式（error）
- 缩进：4空格，禁用制表符，换行缩进与括号对齐
- 行宽：代码≤88字符，注释≤72字符
- 空行：函数/类之间2行，方法间1行，逻辑块间1行
- 导入：分3组（标准/第三方/本地），字母排序，禁止通配导入

### 2. 命名规则（error）
- 变量/函数：小写蛇形（如 user_id、calc_nav()）
- 类/异常：大驼峰（如 DataProcessor、FileError）
- 常量：大写蛇形（如 MAX_RETRY = 3）
- 私有成员：下划线前缀（如 _private_func()）

### 3. 语法规范（error）
- 类型注解：所有函数必须标注参数/返回值类型
- 废弃语法：禁用 pandas.append()、numpy.matrix
- 循环约束：禁止嵌套循环>2层，大数据用 vectorize
- 空值判断：用 is None，禁止 == None

### 4. 注释规范（warning）
- 函数/类：必须加 Google 风格 docstring
- 关键逻辑：加单行注释，禁止无意义注释（如「循环遍历」）
- 复杂计算：分步骤注释（如「1. 计算累计收益 2. 找最大值」）

### 5. 安全约束（critical）
- 禁止硬编码密钥/密码
- 文件读取：校验路径，防止路径遍历
- 网络请求：加超时设置（timeout≤10s）
