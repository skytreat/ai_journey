# 基金推荐系统详细设计文档

## 概述
本设计文档基于现有Python脚本（`fund_recommendation_multiple_factors.py`）和需求文档（`requirements.md`）的分析，针对基金推荐系统进行详细设计。系统旨在通过AKShare库获取中国开放式基金数据，进行多因子分析和推荐，并提供网页界面展示结果。文档涵盖技术选型、数据表Schema设计、数据流、组件图和类图。

## 技术选型

### 数据库
- **选择**: PostgreSQL
- **理由**: 免费、轻量级，支持高级查询（如JOIN、聚合、窗口函数），适合处理基金数据的复杂查询和历史数据存储。相比SQLite，更适合大规模数据和并发访问。
- **版本**: PostgreSQL 15+（支持现代SQL特性）。
- **ORM**: SQLAlchemy（Python原生，便于与Pandas集成）。

### 网页开发
- **后端框架**: Flask
- **理由**: 轻量级Python框架，易于构建API端点和模板，适合快速原型开发。支持与SQLAlchemy集成。
- **前端技术**: React
- **理由**: 提供交互式组件，便于实现动态图表和推荐展示（如使用Plotly.js）。相比纯HTML/JS，更适合复杂UI。
- **其他**: 
  - 图表库: Plotly（用于基金趋势图和热力图）。
  - 样式: Bootstrap（集成到React中）。
  - 部署: 本地Flask开发服务器，或云平台（如Heroku）。

### 其他技术
- **数据获取**: AKShare（现有库，添加随机延迟和并发）。
- **调度**: APScheduler（Python库，用于每日数据更新）。
- **缓存**: 本地文件（PKL格式，扩展为Redis可选）。
- **测试**: pytest（单元测试）。
- **日志**: Python logging模块。
- **配置**: YAML文件（灵活配置API密钥、DB连接等）。

## 数据表Schema设计
基于`requirements.md`中的7个表描述，设计PostgreSQL Schema。假设数据库名为`fund_recommendation`。所有表使用UTF-8编码，支持中文基金名称。

### 1. fund_basic_info（基金基本信息）
```sql
CREATE TABLE fund_basic_info (
    code VARCHAR(10) PRIMARY KEY,  -- 基金代码
    name VARCHAR(100) NOT NULL,     -- 基金名称
    type VARCHAR(50),               -- 基金类型
    establishment_date DATE,        -- 成立日期
    management_company VARCHAR(100) -- 管理公司
);
CREATE INDEX idx_fund_basic_info_type ON fund_basic_info(type);
```

### 2. fund_asset_scale（基金资产规模）
```sql
CREATE TABLE fund_asset_scale (
    code VARCHAR(10) REFERENCES fund_basic_info(code),
    date DATE NOT NULL,
    scale DECIMAL(15,2),  -- 资产规模（亿元）
    PRIMARY KEY (code, date)
);
CREATE INDEX idx_fund_asset_scale_date ON fund_asset_scale(date);
```

### 3. fund_managers（基金经理）
```sql
CREATE TABLE fund_managers (
    code VARCHAR(10) REFERENCES fund_basic_info(code),
    manager_name VARCHAR(50),
    start_date DATE,
    end_date DATE,  -- NULL表示当前经理
    PRIMARY KEY (code, manager_name, start_date)
);
```

### 4. fund_purchase_redemption_status（基金申购赎回状态）
```sql
CREATE TABLE fund_purchase_redemption_status (
    code VARCHAR(10) REFERENCES fund_basic_info(code),
    date DATE NOT NULL,
    purchase_status VARCHAR(20),  -- 申购状态
    redemption_status VARCHAR(20), -- 赎回状态
    PRIMARY KEY (code, date)
);
```

### 5. fund_nav_history（基金净值历史）
```sql
CREATE TABLE fund_nav_history (
    code VARCHAR(10) REFERENCES fund_basic_info(code),
    date DATE NOT NULL,
    nav DECIMAL(10,4),  -- 净值
    accumulated_nav DECIMAL(10,4), -- 累计净值
    PRIMARY KEY (code, date)
);
CREATE INDEX idx_fund_nav_history_date ON fund_nav_history(date);
```

### 6. fund_performance_metrics（基金绩效指标）
```sql
CREATE TABLE fund_performance_metrics (
    code VARCHAR(10) REFERENCES fund_basic_info(code),
    period VARCHAR(20),  -- e.g., '1Y', '3Y'
    annualized_return DECIMAL(5,2),
    max_drawdown DECIMAL(5,2),
    sharpe_ratio DECIMAL(5,2),
    sortino_ratio DECIMAL(5,2),
    calmar_ratio DECIMAL(5,2),
    PRIMARY KEY (code, period)
);
```

### 7. fund_recommendations（基金推荐结果）
```sql
CREATE TABLE fund_recommendations (
    id SERIAL PRIMARY KEY,
    code VARCHAR(10) REFERENCES fund_basic_info(code),
    type VARCHAR(50),
    score DECIMAL(3,1),  -- 5星评分
    rank INTEGER,
    generated_date DATE DEFAULT CURRENT_DATE
);
CREATE INDEX idx_fund_recommendations_type_rank ON fund_recommendations(type, rank);
```

**设计说明**: 
- 外键确保数据一致性。
- 索引优化查询（如按日期或类型过滤）。
- DECIMAL类型用于精确金融数据。
- 约束：非空主键，日期有效性（可添加CHECK约束）。

## 数据流
系统数据流分为三个主要阶段：获取、处理和展示。使用并发和缓存优化性能。

1. **数据获取流**:
   - AKShare API调用 → 随机延迟(0.8-1.5s) → 并发线程池(最多3线程) → 本地缓存检查(每日有效) → 如果缓存失效，插入/更新PostgreSQL表。
   - 错误处理: 重试机制，日志记录失败。

2. **数据处理流**:
   - 从PostgreSQL查询NAV历史 → Pandas DataFrame加载 → 多因子计算(收益率、风险指标) → 加权评分(5星制) → 插入fund_recommendations表 → 生成CSV报告和PNG图表。

3. **数据展示流**:
   - Flask API端点查询recommendations/metrics → 返回JSON → React组件渲染图表/表格 → 用户交互(过滤类型/周期) → 动态更新。

**整体流图** (文本描述):
```
AKShare APIs → [并发获取] → PostgreSQL (插入/更新) → [分析查询] → Pandas处理 → [评分] → 存储结果 → Flask API → React UI
```
- 调度: APScheduler每日触发获取流。
- 缓存: 减少API调用，存储为PKL文件。

## 组件图
系统采用模块化架构，组件间通过数据流连接。

```
[Data Fetcher Module]
    ↓ (API数据)
[Database Layer (PostgreSQL + SQLAlchemy)]
    ↓ (查询结果)
[Analysis Engine (Pandas + 多因子逻辑)]
    ↓ (推荐数据)
[Web Backend (Flask API)]
    ↓ (JSON响应)
[Frontend UI (React + Plotly)]
```

- **Data Fetcher**: 封装AKShare调用，处理并发和缓存。
- **Database Layer**: ORM接口，执行CRUD操作。
- **Analysis Engine**: 核心逻辑，从现有脚本扩展。
- **Web Backend**: RESTful API，提供数据端点。
- **Frontend UI**: 组件化界面，显示图表和列表。

**依赖**: 所有模块依赖Python生态；前端依赖Node.js for React。

## 类图
基于现有代码扩展，添加新类。使用UML风格描述。

```
FundDataFetcher

FundEvaluator
- attributes: df_nav (DataFrame), metrics (dict)
- methods: calculate_returns(), compute_ratios(), score_fund(), generate_report()

DatabaseConnector (new)
- attributes: engine (SQLAlchemy Engine), session
- methods: connect(), insert_data(), query_nav(), close()

WebHandler (new, Flask-based)
- attributes: app (Flask), db_connector
- methods: get_recommendations(), get_metrics(), render_chart()

ReactComponent (new, frontend stub)
- attributes: data (props), chart (Plotly)
- methods: renderTable(), renderChart(), handleFilter()
```

**关系**:
- FundDataFetcher → DatabaseConnector (uses)
- FundEvaluator → DatabaseConnector (queries)
- WebHandler → FundEvaluator (calls for analysis)
- ReactComponent → WebHandler (fetches via API)

**扩展说明**: 新类集成现有逻辑，确保向后兼容。方法签名基于现有代码模式。
